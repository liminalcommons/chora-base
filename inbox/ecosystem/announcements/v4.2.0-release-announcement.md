## What is SAP-001?

SAP-001 Inbox Coordination Protocol is a **formal cross-repository coordination system** for multi-repo ecosystems. It provides:

- **Structured coordination requests** with JSON schema validation
- **Event logging and traceability** (append-only JSONL log)
- **Formalized SLAs** (48h default, 4h urgent response times)
- **5 production CLI tools** for complete automation
- **Capability discovery** via machine-readable CAPABILITIES files

**Use Cases:**
- Cross-repository collaboration (feature requests, bug reports, proposals)
- Formal coordination with SLA commitments
- Ecosystem-wide capability discovery
- AI agent coordination ("inbox status" queries)

## Getting Started

### 1. Install (5 minutes)
```bash
# Clone chora-base (if not already)
git clone https://github.com/liminalcommons/chora-base.git

# Install inbox protocol in your repository
cd your-repo
python ../chora-base/scripts/install-inbox-protocol.py --repo-path .
```

### 2. Verify (2 minutes)
```bash
# Check inbox status
python scripts/inbox-status.py

# Query incoming items (should be empty initially)
python scripts/inbox-query.py --count-by-status
```

### 3. Configure (5 minutes)
Update `inbox/coordination/CAPABILITIES/<your-repo>.yaml` with:
- Repository capabilities (`provides`, `consumes`, `can_receive`)
- Response time SLAs
- Maintainer contact information

### 4. Start Coordinating
```bash
# Send coordination request to another repository
python scripts/generate-coordination-request.py

# Respond to incoming requests
python scripts/respond-to-coordination.py COORD-123 accepted
```
