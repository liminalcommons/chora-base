## Methodology Improvements

### Improvement #1: Pre-Flight SAP Inclusion Check

**What**: Always verify actual file generation before planning verification method

**Why**: Some SAPs marked as "incremental" may still be generated by fast-setup

**Example**: SAP-009 (`"included_by_default": false`) but AGENTS.md/CLAUDE.md always generated

**Application**:
```bash
# Before planning verification, check if files exist
ls -la AGENTS.md CLAUDE.md
# If they exist, verification method = fast-setup (verify existing files)
# If missing, verification method = incremental adoption
```

**Impact**: Saves time, ensures correct verification approach

### Improvement #2: Template-Based Documentation Creation

**What**: Use static-template/ files as starting point for SAP adoption

**Evidence**: DOCUMENTATION_STANDARD.md copied in 1 minute, saved significant writing time

**Application**: For any SAP with template files:
1. Check static-template/ for existing files
2. Copy template to project
3. Customize placeholders
4. Verify criteria

**Impact**: Faster adoption, higher consistency
